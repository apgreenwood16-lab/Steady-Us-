# START HERE — Which SteadyUs file to use when

This guide is the simplest way to navigate the SteadyUs materials.

The confusion comes from the fact that there are **two different kinds of materials**. The first kind is **sales material**, which helps you explain or sell the offer. The second kind is **delivery material**, which helps you actually run the offer once a couple is engaging with it.

There is also a smaller third category: **handover/reference material**. Those files are not for clients. They exist so you, or someone helping you later, can understand how the project was built.

## 1. The quickest answer

If you want the shortest possible version, use this table first.

| If you want to... | Use this file / deck | Why this is the right one |
| --- | --- | --- |
| Explain the full SteadyUs offer | **SteadyUs — Couples Budget Reset** (main package deck) | This is the full sales deck. It explains the offer, what is included, who it is for, and the price. |
| Give someone a shorter intro to the offer | **SteadyUs — Prospect Deck** | This is the shorter, lighter sales version. |
| Show the offer on one page only | **Couples Budget Reset at a glance** | This is the compressed one-slide summary. |
| Help a client understand what will happen in the sessions | **SteadyUs Client Session Guide** (PDF) | This is the practical client-facing delivery guide. |
| Run the sessions consistently yourself | **SteadyUs Coach Delivery Playbook** (PDF) | This is the internal facilitation guide. |
| Take structured notes during or after a session | **SteadyUs Coach Call Notes Form** (PDF) | This is the working form for capturing what happened and what comes next. |
| Edit the wording behind the delivery materials | `CLIENT_SESSION_CHECKLIST.md`, `SESSION_DELIVERY_PLAN.md`, `COACH_CALL_NOTES_TEMPLATE.md` | These are the source text documents that sit behind the polished delivery PDFs. |
| Hand the project to someone else or rebuild it later | `README.md`, `CLAUDE_PASTE_READY_HANDOVER.md`, `WEBSITE_REBUILD_NOTES.md`, `PRESENTATIONS_REBUILD.md` | These are reference and handover files, not live client or delivery assets. |

## 2. Sales materials

These are the materials you use when you are **explaining, introducing, or selling** the offer.

| Material | Main role | Best moment to use it | Do not use it for |
| --- | --- | --- | --- |
| **SteadyUs — Couples Budget Reset** (main package deck) | Full sales presentation | Use when someone wants the complete explanation of the offer. | Do not use it as a session-delivery guide. |
| **SteadyUs — Prospect Deck** | Short prospect introduction | Use when you want a quicker, lighter version of the pitch. | Do not use it when you need full delivery detail. |
| **Couples Budget Reset at a glance** | One-page summary | Use when you need a very short visual summary. | Do not use it as the main explanation of the service. |

## 3. Delivery materials

These are the materials you use when you are **delivering the service**, not selling it.

| Material | Main role | Best moment to use it | Do not use it for |
| --- | --- | --- | --- |
| **SteadyUs — Couples Budget Reset** (client session deck) | Client-facing session walkthrough | Use before or during a client-facing conversation when you want to explain the process clearly. | Do not use it as your internal facilitation manual. |
| **SteadyUs — Coach Delivery Guide** (coach deck) | Internal live facilitation deck | Use when you want a visual internal guide for how to run the sessions. | Do not send it to clients as their main guide. |
| **SteadyUs Client Session Guide** (PDF) | Client handout / client orientation | Use when sending clients something practical they can read, save, or print. | Do not use it as your internal coach notes tool. |
| **SteadyUs Coach Delivery Playbook** (PDF) | Internal written delivery guide | Use before sessions, while training another coach, or when checking consistency. | Do not use it as a client handout. |
| **SteadyUs Coach Call Notes Form** (PDF) | Internal working notes sheet | Use during and after sessions to capture patterns, decisions, and next steps. | Do not use it as a sales document. |

## 4. Source documents behind the polished delivery files

These three Markdown documents are important, but they are **source material**, not the most usable day-to-day versions.

| Source file | What it is | What it turns into |
| --- | --- | --- |
| `CLIENT_SESSION_CHECKLIST.md` | Original written client guidance | Basis for the **SteadyUs Client Session Guide** PDF |
| `SESSION_DELIVERY_PLAN.md` | Original internal delivery playbook | Basis for the **SteadyUs Coach Delivery Playbook** PDF |
| `COACH_CALL_NOTES_TEMPLATE.md` | Original internal notes structure | Basis for the **SteadyUs Coach Call Notes Form** PDF |

This is where some of the overlap comes from. The Markdown files and the PDFs are not doing different jobs. The PDFs are simply the more usable versions of the same delivery content.

## 5. Handover and reference materials

These files are mainly for continuity, transfer, and rebuilding.

| File | Role | When to use it |
| --- | --- | --- |
| `README.md` | Top-level orientation file | Use when you need to understand the backup structure. |
| `CLAUDE_PASTE_READY_HANDOVER.md` | AI handover summary | Use when handing the project to Claude or another collaborator. |
| `WEBSITE_REBUILD_NOTES.md` | Website rebuild reference | Use when recreating or checking the web build. |
| `PRESENTATIONS_REBUILD.md` | Deck rebuild reference | Use when tracing how the deck assets were rebuilt. |

## 6. The cleanest way to think about all of this

If you want the simplest mental model, use this one sentence:

> **Sales decks explain the offer, delivery files run the offer, and handover docs explain the project itself.**

And if you want the minimum set to actually operate SteadyUs, it is probably these five items:

| Minimum working set | Why it matters |
| --- | --- |
| Main package deck | Full sales conversation |
| Prospect deck | Short sales conversation |
| Client session guide PDF | Client-facing delivery support |
| Coach delivery playbook PDF | Internal delivery consistency |
| Coach call notes form PDF | Internal session record-keeping |
