# START HERE — Which SteadyUs file to use when

This guide is the simplest way to navigate the SteadyUs materials.

The confusion comes from the fact that there are **two different kinds of materials**. The first kind is **sales material**, which helps you explain or sell the offer. The second kind is **delivery material**, which helps you actually run the offer once a couple is engaging with it. There is also a third category, **handover/reference material**, which exists mainly so you or someone helping you later can understand how the project was built.

## 1. Quick decision tree

Use this first if you just want the fastest answer.

| Question | If yes, use this | If no, go to |
| --- | --- | --- |
| Are you trying to **sell or explain the offer**? | Use one of the **sales decks** below. | Go to the next question. |
| Are you trying to **help a client understand the session process**? | `CLIENT DELIVERY - SteadyUs Client Session Guide.pdf` | Go to the next question. |
| Are you trying to **run the sessions consistently yourself**? | `INTERNAL ONLY - SteadyUs Coach Delivery Playbook.pdf` | Go to the next question. |
| Are you trying to **take notes during or after a session**? | `INTERNAL ONLY - SteadyUs Coach Call Notes Form.pdf` | Go to the next question. |
| Are you trying to **edit the underlying wording** behind the delivery files? | Use the labeled **SOURCE** Markdown files. | Go to the next question. |
| Are you trying to **understand, transfer, or rebuild the project**? | Use the handover/reference docs. | You probably need one of the items in the tables below. |

## 3. Sales materials

These are the materials you use when you are **explaining, introducing, or selling** the offer.

| Material | Main role | Best moment to use it | Do not use it for |
| --- | --- | --- | --- |
| `SALES - SteadyUs Full Offer Deck.pdf` | Full sales presentation | Use when someone wants the complete explanation of the offer. | Do not use it as a session-delivery guide. |
| `SALES - SteadyUs Prospect Deck.pdf` | Short prospect introduction | Use when you want a quicker, lighter version of the pitch. | Do not use it when you need full delivery detail. |
| **Couples Budget Reset at a glance** | One-page summary | Use when you need a very short visual summary. | Do not use it as the main explanation of the service. |

## 4. Delivery materials

These are the materials you use when you are **delivering the service**, not selling it.

| Material | Main role | Best moment to use it | Do not use it for |
| --- | --- | --- | --- |
| **SteadyUs — Couples Budget Reset** (client session deck) | Client-facing session walkthrough | Use before or during a client-facing conversation when you want to explain the process clearly. | Do not use it as your internal facilitation manual. |
| **SteadyUs — Coach Delivery Guide** (coach deck) | Internal live facilitation deck | Use when you want a visual internal guide for how to run the sessions. | Do not send it to clients as their main guide. |
| `CLIENT DELIVERY - SteadyUs Client Session Guide.pdf` | Client handout / client orientation | Use when sending clients something practical they can read, save, or print. | Do not use it as your internal coach notes tool. |
| `INTERNAL ONLY - SteadyUs Coach Delivery Playbook.pdf` | Internal written delivery guide | Use before sessions, while training another coach, or when checking consistency. | Do not use it as a client handout. |
| `INTERNAL ONLY - SteadyUs Coach Call Notes Form.pdf` | Internal working notes sheet | Use during and after sessions to capture patterns, decisions, and next steps. | Do not use it as a sales document. |

## 5. Source documents behind the polished delivery files

These three Markdown documents are important, but they are **source material**, not the most usable day-to-day versions.

| Source file | What it is | What it turns into |
| --- | --- | --- |
| `CLIENT DELIVERY - SOURCE - Client Session Checklist.md` | Original written client guidance | Basis for the **client session guide** PDF |
| `INTERNAL ONLY - SOURCE - Session Delivery Plan.md` | Original internal delivery playbook | Basis for the **coach delivery playbook** PDF |
| `INTERNAL ONLY - SOURCE - Coach Call Notes Template.md` | Original internal notes structure | Basis for the **coach call notes form** PDF |

This is where some of the overlap comes from. The Markdown files and the PDFs are not doing different jobs. The PDFs are simply the more usable versions of the same delivery content.

## 6. Handover and reference materials

These files are mainly for continuity, transfer, and rebuilding.

| File | Role | When to use it |
| --- | --- | --- |
| `README.md` | Top-level orientation file | Use when you need to understand the backup structure. |

| `WEBSITE_REBUILD_NOTES.md` | Website rebuild reference | Use when recreating or checking the web build. |

## 7. The cleanest way to think about all of this

If you want the simplest mental model, use this one sentence:

> **Sales decks explain the offer, delivery files run the offer, and handover docs explain the project itself.**

And if you want the minimum set to actually operate SteadyUs, it is probably these five items:

| Minimum working set | Why it matters |
| --- | --- |
| `SALES - SteadyUs Full Offer Deck.pdf` | Full sales conversation |
